## 🎯 HOW TO USE THE MODEL

### 1️⃣ Frequency Prediction for a New Company

**Input Information Needed:**
```
- Company Annual Revenue (USD)
- Number of Employees
- Public Company Status (Yes/No)
```

**Prediction Steps:**

```python
import numpy as np

# Company data
revenue = 50_000_000_000  # $50B
employees = 100_000
is_public = 1  # 1=Yes, 0=No

# Feature transformation
log_revenue = np.log1p(revenue)
log_employees = np.log1p(employees)
company_size = 3  # Enterprise (coded 0-3)

# Model coefficients (from poisson_frequency_model)
beta = {
    'intercept': -0.3326,
    'log_revenue': 0.0478,
    'log_employees': -0.0512,
    'is_public': -0.0094,
    'size': -0.0405
}

# Calculate prediction
log_lambda = (beta['intercept'] + 
              beta['log_revenue'] * log_revenue + 
              beta['log_employees'] * log_employees + 
              beta['is_public'] * is_public + 
              beta['size'] * company_size)

predicted_frequency = np.exp(log_lambda)

# Expected: ~1.20-1.25 incidents over analysis period
print(f"Predicted frequency: {predicted_frequency:.3f}")
```

### 2️⃣ Risk Scoring

**Formula:**
```
Risk Score = (Predicted Frequency / Mean Frequency) × 100
Risk Score = (Predicted Frequency / 1.1354) × 100
```

**Interpretation:**
```
Risk Score 85-95   = Below-average risk (lower premium loading)
Risk Score 95-105  = Average risk (standard pricing)
Risk Score 105-115 = Above-average risk (higher premium loading)
```

### 3️⃣ Premium Calculation (Framework)

**Full Premium Formula:**
```
Annual Premium = Base Pure Premium × Frequency RIF × Severity RIF × Expense Loading

Where:
  Base Pure Premium = Industry benchmark (to be determined)
  Frequency RIF = Risk Score / 100 (from model)
  Severity RIF = Company-specific loss factor (severity model needed)
  Expense Loading = 1 + (expenses + profit margin + uncertainty)
```

**Simplified Example:**
```
Given:
  Base Pure Premium = $100,000
  Company Risk Score = 108 (Frequency RIF = 1.08)
  Severity RIF = 1.10 (from severity model)
  Expense Loading = 1.35 (35% loading)

Adjusted Premium = $100,000 × 1.08 × 1.10 × 1.35 = $161,010
```

## ⚙️ MODEL SPECIFICATIONS

### Mathematical Definition

**Poisson GLM with Log Link:**
```
E[Y|X] = λ = exp(β₀ + β₁X₁ + β₂X₂ + ... + βₚXₚ)

Where:
  Y = Incident frequency (count data)
  λ = Expected frequency (rate parameter)
  X = Feature vector [log_revenue, log_employees, is_public, size_encoded]
  β = Fitted coefficients
```

### Fitted Coefficients

| Feature | Coefficient | Rate Ratio* |
|---------|-------------|------------|
| Intercept | -0.3326 | — |
| log_revenue | +0.0478 | 1.0049 |
| log_employees | -0.0512 | 0.9950 |
| is_public_company | -0.0094 | 0.9906 |
| company_size_encoded | -0.0405 | 0.9603 |

*Rate Ratio = exp(coefficient) = multiplicative effect of 1-unit increase

### Regularization Details

- **Type:** Ridge (L2) - prevents overfitting while preserving all features
- **Optimal Parameter:** α = 0.1292
- **Method:** 5-fold cross-validation with grid search
- **Convergence:** Achieved at 342 iterations

## 🚨 MODEL LIMITATIONS & CAVEATS

### Known Limitations

1. **Reporting Bias:** Private companies may underreport incidents
2. **Limited Time Window:** Only 5 years of historical data
3. **Omitted Variables:** Industry not directly included
4. **Linearity:** Log-linear assumption may not capture interactions
5. **Tail Risk:** Underestimates very high-frequency companies

### When NOT to Use This Model

❌ For companies with <$1M revenue (extrapolation)
❌ For newly formed companies (<1 year old)
❌ For companies outside 38 countries in database
❌ For incident predictions >1 year forward
❌ For individual incident severity prediction

### Mitigation Strategies

1. Add industry dummy variables (expand model)
2. Incorporate temporal trends
3. Blend model with expert judgment for edge cases
4. Use credibility theory for sparse data
5. Annual model retraining with new incidents

## 🎓 NEXT PHASE: SEVERITY MODELING

### Objective
Build loss distribution model to estimate E[Loss | Company Characteristics]

### Data Source
`financial_impact_1.csv` - 778 incident records with loss amounts

### Candidate Distributions

**Option 1: Lognormal Distribution**
```
Pros: Flexible, handles skewed losses, natural for financial data
Cons: May underestimate tail risk
Formula: ln(Loss) ~ N(μ, σ²)
```

**Option 2: Pareto Distribution**
```
Pros: Better tail modeling, realistic for cyber losses
Cons: Less flexible for middle-range losses
Formula: P(Loss > x) = (x_m/x)^α
```

**Option 3: Gamma Distribution**
```
Pros: Flexible shape, good for operational data
Cons: Limited tail behavior
Formula: Loss ~ Gamma(α, β)
```

### Recommended Approach

1. Fit all three distributions to historical loss data
2. Test goodness-of-fit (Anderson-Darling, Kolmogorov-Smirnov)
3. Compare AIC/BIC for model selection
4. Separately model large losses (>$10M) for tail risk
5. Develop loss ratio model: E[Loss | Revenue, Industry, Attack Vector]
